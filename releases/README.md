# Release Records

This directory tracks reviewed provider submissions.

Create one release manifest per provider submission from `release-manifest.template.json`. Each completed manifest should record the provider, artifact path, version tag, submitted commit SHA, reviewers, marketplace contact, launch date, and any submission notes.

Use `SUBMISSION_CHECKLIST.md` before each provider submission to confirm package, OAuth, copy, security, and release-record requirements.

Do not record a release manifest until the artifact has completed Product, Content Design, Legal, Security, Analytics, and Engineering review.
